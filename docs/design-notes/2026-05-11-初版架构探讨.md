# 初版架构探讨记录

> 日期：2026-05-11 | 参与：用户 + Claude Code

## 关键决策

1. **技术路线**：Electron + React/Vue + Capacitor（移动端），共享核心代码
2. **数据同步**：本地为主 + 可选云同步（WebDAV/云盘等适配器）
3. **MVP边界**：桌面端优先，移动端留到v1.0
4. **核心差异化**：智能交流中枢（朋友式聊天替代表单填写）
5. **设计哲学**：动态适配而非固定模板

## 章纲要协作模式

最终确定的双向自由入口 + 三向写作入口模式：
- 章纲要：用户自写 / AI生成草案，两条路径汇合后经审核锁定
- 正文：用户自写 / AI生成草案 / 混合模式，三向自由
- 任何环节用户都有选择权，AI履行审核+建议职责

## 竞品调研发现

- InkOS（6k⭐）：最成熟的多Agent架构，33维度审计，7个Truth Files
- AI-Novel-Writing-Assistant（1.2k⭐）：Creative Hub，LangGraph Agent，写法引擎
- 51mazi/我要码字（282⭐）：最接近目标的Electron+Vue桌面应用
- NovelClaw（304⭐）：动态记忆优先，可编辑记忆银行
- Vela（196⭐）：BYOK模式，四分屏布局，本地RAG
